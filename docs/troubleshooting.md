# 故障排查

## 标签显示为无

通常是因为 Crawler 默认没有抓标签。

这是预期行为。标签需要额外请求，会更慢，也更容易触发风控。Planner 仍会使用标题、UP、来源夹、分区和简介分类。

需要标签时，勾选 `抓标签（更慢，审核更准）` 后重新采集。

## 采集器只下载一个文件

这是现在的默认设计。

默认只下载：

```text
bilibili-favorites-evidence.jsonl
```

旧版本会同时下载 inventory 和 summary，后来为了减少干扰和 Chrome 多文件权限提示，改成默认单文件。

## 暂停后进度会丢吗

不会。

`暂停` 会把检查点保存到浏览器 `localStorage`。回到同一个 B 站收藏夹页面后，点击 `继续采集` 即可。

`取消重来` 才会清除检查点并从 0 开始。

## 出现 403、412、验证码、登录失效

先停下，不要连续重试。

建议：

- 确认 Chrome 已登录 B 站。
- 刷新收藏夹页面。
- 关闭抓标签。
- 晚点再试。
- 先用 `测试采集` 验证。

## 复核页数据太多

使用：

- 搜索
- 状态筛选
- 分页
- 批量主分类
- 批量范围状态勾选

默认批量范围只处理 `待复核`，不会覆盖已通过行。

## 全部批准后页面空了

如果当前状态筛选是 `待复核`，批准后筛选结果会变空。切换到 `全部状态` 即可。

新版复核页会尽量在批准当前页后避免卡在空筛选状态。

## Planner 无法生成任务包

常见原因：

- 没有 `approved` 行。
- 已通过行缺少 `aid`。
- 已通过行缺少来源夹 id。
- 已通过行缺少主分类。
- JSONL 某行格式错误。

把 `reviewed-classification.jsonl` 交给 Planner auto 入口，看校验错误即可。

## Executor 提示目标夹 ID 缺失

使用 Executor 的目标夹创建/回填功能。它会先匹配已有目标夹，再创建真正缺失的目标夹。

不要手工乱填 `folder_id`。

## Executor 出现中间态

move 是两步：

1. 添加到目标夹。
2. 从来源夹移出。

中途暂停或失败时可能出现中间态。使用 Executor 恢复执行即可。
