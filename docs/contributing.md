# 贡献指南

贡献时请保持组件边界清晰。

## 基本规则

- Crawler 必须保持只读。
- Planner 必须保持本地文件处理。
- 复核页不能写 B 站。
- Executor 必须保留暂停、恢复和保守节奏。
- 不添加凭据收集。
- 不添加隐藏自动批准。
- 不把删除收藏夹、清理失效视频等维护操作塞进迁移任务包。

## 适合优先贡献的内容

- 文档修正。
- 更多故障排查案例。
- 复核页易用性改进。
- 更清晰的校验错误。
- 更保守的分类规则。
- 脱敏假数据测试样例。

## 测试要求

Crawler / Planner：

- `node --check`
- 样例 JSONL 校验。
- 修改 Skill 文件时跑 quick validate。

Executor：

- `node --check`
- 静态契约测试。
- UI 改动需要浏览器手动冒烟验证。

## 安全报告

不要在公开 issue 中贴真实 Cookie、SESSDATA、csrf、个人收藏夹列表或真实任务包。请使用脱敏样例描述问题。
