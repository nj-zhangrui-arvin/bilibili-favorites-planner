# 人工复核

复核页是分类建议和真实执行之间的质量关口。

## 输入输出

输入：

- `classification-review.jsonl`

输出：

- `reviewed-classification.jsonl`

复核页是本地静态 HTML，不需要后端服务。

## 状态含义

| 状态 | 含义 | 是否进入任务包 |
| --- | --- | --- |
| `approved` | 人工确认可执行 | 是 |
| `needs_review` | 需要人工判断 | 否 |
| `untrackable` | 视频失效或缺少可执行信息 | 否 |
| `skip` | 人工决定跳过 | 否 |

## 可编辑字段

复核页只应该修改：

- `review_status`
- `reviewed_primary_folder`
- `reviewed_secondary_folders`

Executor 只信任这些人工复核字段。

## 搜索范围

搜索会匹配：

- 标题
- UP
- BV 号
- 来源收藏夹
- 标签
- 建议分类
- 人工分类

占位建议主分类 `待复核` 不作为有效搜索命中。

## 批量分类

批量主分类有两层范围：

1. 当前搜索和状态筛选结果。
2. 批量范围状态勾选。

默认只勾选 `待复核`，避免误覆盖已经通过的行。

## 标签为什么可能是无

如果 Crawler 没勾选 `抓标签（更慢，审核更准）`，复核页会显示 `标签：无`。

这不是错误。Planner 仍会使用标题、UP、来源夹、分区和简介分类。需要标签辅助判断时，重新勾选抓标签再采集。

## 分页和草稿

分页不会丢失已编辑分类。复核页会把草稿保存到浏览器本地。导出前确认统计和当前筛选状态符合预期。
