# 安全模型

本项目围绕“人工复核”和“保守自动化”设计。

## 组件边界

| 组件 | 读 B 站 | 写 B 站 | 本地状态 |
| --- | --- | --- | --- |
| Crawler | 是 | 否 | 是 |
| Planner | 否 | 否 | 文件 |
| Review Page | 否 | 否 | 浏览器草稿 |
| Executor | 是 | 是 | 是 |

## 凭据处理

工具不会要求用户粘贴：

- Cookie
- SESSDATA
- csrf
- 密码

浏览器脚本只在已登录的 B 站页面上下文中发请求。

## Crawler 风险控制

- 只调用读取接口。
- 默认不抓标签，减少请求量。
- 慢速请求。
- 支持暂停和继续。
- 遇到登录、验证码、403、412 等风险信号会停止。
- 默认只下载一个 evidence 文件。

## Planner 风险控制

- 只处理本地文件。
- 不访问 B 站。
- 不确定的视频进入 `needs_review`。
- 只有 `approved` 行能生成任务。
- 校验复核文件和任务包。

## Executor 风险控制

- 只导入明确的任务包。
- 缺失目标夹需要用户操作创建。
- move 分两步：先添加到目标夹，再移出来源夹。
- 支持中间态恢复。
- 遇到 403、412、登录失效、csrf 缺失或写接口失败会停止。
- 可导出备份和执行报告。

## 人工复核要求

Planner 建议不能直接执行。

用户必须在复核页确认分类并导出 `reviewed-classification.jsonl`。保留在 `needs_review`、`untrackable`、`skip` 的行不会进入执行任务包。

## 平台边界

仅用于管理自己账号下的收藏夹。不要用于第三方账号、商业抓取、刷量、绕过平台限制或影响 B 站服务稳定性。
